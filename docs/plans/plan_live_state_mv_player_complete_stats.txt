4) Proposed Data Model
4.1 New table: live_match_state
- event_id BIGINT NOT NULL
- league_id TEXT NOT NULL
- home_team TEXT
- away_team TEXT
- home_score INTEGER DEFAULT 0
- away_score INTEGER DEFAULT 0
- status TEXT
- minute INTEGER DEFAULT 0
- poll_count INTEGER DEFAULT 0
- insight_text TEXT
- stats_core_json JSONB
  (chi giu subset quan trong cho UI nhanh: possession, shots_on_target,
   xg neu co, dangerous_attacks neu co)
- last_processed_seq BIGINT
  (duoc scheduler update: seq cuoi cung da persist vao live_match_state,
   de detect drift va resume worker)
- last_published_seq BIGINT
  (optional, duoc API layer update neu can; neu khong dung thi de NULL,
   KHONG duoc scheduler ghi vao cot nay)
- loaded_at TIMESTAMPTZ DEFAULT NOW()
- updated_at TIMESTAMPTZ

Constraints:
- PRIMARY KEY (event_id)
- CONSTRAINT fk_lms_event
    FOREIGN KEY (event_id, league_id)
    REFERENCES ss_events (event_id, league_id)
    ON DELETE CASCADE
    DEFERRABLE INITIALLY DEFERRED
- CONSTRAINT chk_lms_stats_core_json_object
  CHECK (stats_core_json IS NULL OR jsonb_typeof(stats_core_json) = 'object')

Indexes:
- idx_lms_status ON live_match_state(status)
- idx_lms_league_status ON live_match_state(league_id, status)

4.2 Change table: live_incidents
- ADD COLUMN seq BIGSERIAL
Indexes:
- idx_live_inc_event_seq ON live_incidents(event_id, seq)

4.3 New MV: mv_player_complete_stats
Grain:
- 1 row / (player_id, league_id, season)

Core join rules (must-follow):
- Season mapping for shots:
  shots.season = SPLIT_PART(pss.season, '-', 1)::INTEGER
  (example: pss.season='2024-2025' -> 2024)
- Goalkeeper join (mandatory season-safe join):
  LEFT JOIN gk_stats gk
    ON gk.player_id = pss.player_id
   AND gk.league_id = pss.league_id
   AND gk.season = pss.season
  Note: This requires gk_stats PK/grain to include season first.
- xg_overperformance formula:
  goals_from_shots - total_xg
  (positive = overperform, negative = underperform)

Columns (v1):
Identity/profile:
- player_id, player_name, team_id, league_id, season, position, nationality
- player_image_url, market_value_numeric

Attacking/core:
- minutes_90s, goals, assists, goals_per90, shots, shots_on_target_pct

Understat shot context:
- has_understat_crossref
- total_xg
- xg_per90
- goals_from_shots
- xg_overperformance

Defensive:
- tackles, interceptions, blocks, clearances (neu co), pressures (neu co)

Possession/progression:
- progressive_carries, progressive_passes_received, take_ons_won_pct

Goalkeeper subset:
- gk_save_pct, gk_clean_sheets (nullable, chi co gia tri voi GK)

Metadata:
- built_at

NULL / COALESCE policy (strict):
- Counting stats from base FBref table (goals, assists, shots, minutes_90s):
  keep source value; do not force 0 if source NULL.
- Understat crossref semantics:
  - has_understat_crossref = TRUE when player_crossref.understat_player_id exists.
  - has_understat_crossref = FALSE when no understat mapping exists.
- Understat aggregate fields:
  - if has_understat_crossref = FALSE: total_xg, goals_from_shots, xg_per90,
    xg_overperformance MUST be NULL (unknown, not zero).
  - if has_understat_crossref = TRUE and player has no shot rows:
    total_xg = 0, goals_from_shots = 0, xg_overperformance = 0.
  - xg_per90 stays NULL when minutes_90s is NULL or 0.
- Defensive/possession optional joins:
  keep NULL when row missing (do not COALESCE to 0 at MV layer).
- GK fields:
  keep NULL for non-GK players (never coalesce to 0).

built_at semantics:
- built_at = NOW() evaluated at MV REFRESH time (snapshot freshness marker).

Indexes:
- UNIQUE (player_id, league_id, season)
- idx_mvpcs_league_season (league_id, season)
- idx_mvpcs_team (team_id, league_id, season)

7) Migration & Rollout (no downtime)
Step 0: Prepare
- Deploy schema changes only.
- No reader switch.

Step 1: Dual-write
- scheduler_master write both live_snapshots + live_match_state.
- Keep all current readers unchanged.
- Observe 3-5 days.
- Drift guard in live loop:
  - compare key fields every 5 minutes while match status='inprogress'
  - mandatory compare trigger at status transition to 'finished'
  - daily check remains historical reconcile only
  - alert mechanism on mismatch:
    - write structured ERROR log
    - increment metric live_drift_mismatch_count
    - DO NOT raise exception that stops polling loop

Step 2: Reader migration
- Move low-latency readers to live_match_state first.
- Incident stream move to seq cursor.
- Keep fallback to live_snapshots for one release window.

Step 3: MV adoption
- Create mv_player_complete_stats and refresh nightly.
- Switch player comparison queries to MV.
- Keep old query behind feature flag/fallback.
- Milestone gate: Step 3 blocked until Phase 0 (gk_stats season-safe) is complete.

Step 4: Cleanup
- Optional: stop writing heavy JSON fields in live_snapshots
  after all readers migrated.
- Review last_published_seq usage at Milestone 3:
  - if still unused (NULL-only), schedule DROP COLUMN in next schema cycle.

8) Validation Checklist
Item 3:
- live_match_state row count ~ active/known events (no missing event in dual-write).
- max lag (live_match_state.loaded_at vs live_snapshots.loaded_at) <= 3s p95.
- incident seq monotonic per event.
- no duplicate incidents after seq introduction.
- FK validation: no orphan row in live_match_state against ss_events.
- Semantics validation:
  - scheduler updates only last_processed_seq
  - scheduler never writes last_published_seq
- Real-time drift validation:
  - in-progress diff check every 5 minutes
  - forced diff check at match finish
  - alert immediately on mismatch in score/status/minute
  - polling loop remains alive after alert (no hard-fail)

Seq migration note:
- After ADD COLUMN seq BIGSERIAL on existing data, historical incidents receive
  generated seq values that may not reflect original insertion order.
- seq is authoritative cursor for matches ingested after migration; historical
  replay ordering should not depend on seq.

Item 4:
- mv row count matches distinct (player_id, league_id, season) from base table.
- random 30 players manual reconcile vs legacy joins with strict tolerance policy:
  - counting stats (goals, assists, cards): exact match (tolerance = 0)
  - float stats (total_xg, xg_per90): abs diff <= 0.001
  - GK nullable stats: NULL for non-GK is expected (not error)
- query latency comparison endpoint reduced (target >= 40%).
- GK fanout guard:
  - assert max joined gk row count per (player_id, league_id, season) = 1

10) Risks & Mitigations
Risk A: Field mismatch between sources in mv_player_complete_stats.
- Mitigation:
  - enforce season cast rule via SPLIT_PART(...)
  - enforce NULL/COALESCE policy from section 4.3
  - add validation SQL sample set by league/season

Risk B: Dual-write drift.
- Mitigation:
  - live check every 5 minutes during inprogress matches
  - mandatory check on finished transition
  - daily diff kept only for historical reconcile
  - emit ERROR log + metric, do not crash poll loop

Risk C: REFRESH CONCURRENTLY lock/contention.
- Mitigation: keep unique index, schedule off-peak, fallback to non-concurrent
  only in maintenance window 03:00-04:00 UTC+7.
